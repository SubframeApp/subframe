Subframe works with any framework using TypeScript, React, and Tailwind CSS. The CLI configures your project automatically to sync your components and theme.

## Prerequisites

- **TypeScript** — Subframe components are fully typed
- **React** (16.8+) — Hooks support required
- **Tailwind CSS** (v3 or v4) — For component styling

<Note>Subframe works with Next.js, Vite, Astro, Remix, and other React frameworks.</Note>

## Install

Run the init command in your repository root:

<CodeGroup>
  ```bash npm
  npx @subframe/cli@latest init
  ```
  ```bash yarn
  yarn dlx @subframe/cli@latest init
  ```
  ```bash pnpm
  pnpx @subframe/cli@latest init
  ```
  ```bash bun
  bunx @subframe/cli@latest init
  ```
</CodeGroup>

### Authenticate

Paste your access token from [app.subframe.com/cli/auth](https://app.subframe.com/cli/auth):

```
> No existing credentials found.
> To get new credentials, please visit the following URL in your web browser:
> https://app.subframe.com/cli/auth

? Access token ›
```

### Configure

Answer configuration questions based on your codebase. Press Enter for defaults:

```
Where should the Subframe components be synced to? › ./src
Configure an alias for the subframe component directory (e.g. @/ui) › @/ui/*
Do you want Subframe to configure your Tailwind config? › (Y/n)
Would you like to sync all of your Subframe components? › (Y/n)
Would you like to install dependencies? › (Y/n)
Would you like to start the dev server? › (Y/n)
```

The CLI syncs components, configures Tailwind with theme tokens, installs dependencies, and optionally starts your dev server.

## Verify installation

Test that Subframe works:

```tsx App.tsx
import { Button } from "@/ui/Button";

export default function App() {
  return <Button>Click me</Button>;
}
```

If the button renders with styling, you're set!

## Common issues

<AccordionGroup>
<Accordion title="Styles not applying">

If components render without styling:

1. Verify Tailwind config includes Subframe theme preset
2. Check content paths include Subframe component directory
3. Ensure Tailwind processes your CSS correctly

See [Tailwind config](/developing/tailwind-config) for details.

</Accordion>
<Accordion title="Import errors">

If you get import errors:

1. Check `tsconfig.json` alias matches your configuration
2. Verify components synced by checking your component directory
3. Run `npx @subframe/cli sync --all` to re-sync

</Accordion>
<Accordion title="TypeScript errors">

If you see TypeScript errors in components:

1. Ensure your project uses TypeScript (not JavaScript)
2. Verify `@types/react` is installed
3. Check `tsconfig.json` includes the component directory

</Accordion>
</AccordionGroup>
