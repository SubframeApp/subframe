---

Install Subframe manually when the CLI doesn't work with your setup or you need custom configuration.

<Note>
This guide assumes you have TypeScript, React, and Tailwind CSS already configured. The CLI automates these steps and handles edge casesâ€”use manual installation only when necessary.
</Note>

## Step 1: Install dependencies

<CodeGroup>
  ```bash npm
  npm install @subframe/core
  ```
  ```bash yarn
  yarn add @subframe/core
  ```
  ```bash pnpm
  pnpm add @subframe/core
  ```
  ```bash bun
  bun add @subframe/core
  ```
</CodeGroup>

## Step 2: Create directory structure

Create directories for Subframe:

```bash
mkdir -p src/ui
mkdir -p .subframe
```

## Step 3: Create sync configuration

Create `.subframe/sync.json` with your project settings:

```json .subframe/sync.json
{
  "directory": "./src/ui",
  "importAlias": "@/ui/*",
  "projectId": "your-project-id"
}
```

Find your project ID in your Subframe app URL: `app.subframe.com/projects/{projectId}`

## Step 4: Configure TypeScript

Add path aliases to `tsconfig.json`:

```json tsconfig.json
{
  "compilerOptions": {
    "baseUrl": ".",
    "paths": {
      "@/ui/*": ["./src/ui/*"]
    }
  }
}
```

## Step 5: Get your theme configuration

### For Tailwind v3

Copy your theme from Subframe's Code mode and create `src/ui/tailwind.config.js`:

```js src/ui/tailwind.config.js
module.exports = {
  theme: {
    extend: {
      colors: {
        brand: {
          50: "#...",
          // ... your brand colors
        },
        neutral: {
          // ... your neutral colors
        },
        // ... other colors
      },
      fontFamily: {
        sans: ["Inter", "sans-serif"],
        // ... other fonts
      },
      fontSize: {
        // ... font sizes from your theme
      },
      // ... other theme tokens
    },
  },
}
```

### For Tailwind v4

Copy your theme from Subframe and create `src/ui/theme.css`:

```css src/ui/theme.css
@import "tailwindcss";

@theme {
  --color-brand-50: #...;
  --color-brand-100: #...;
  /* ... other custom properties */
}
```

## Step 6: Configure Tailwind in your project

### For Tailwind v3

Update your root `tailwind.config.js`:

```js tailwind.config.js
module.exports = {
  presets: [require("./src/ui/tailwind.config.js")],
  content: [
    "./src/**/*.{js,jsx,ts,tsx}",
    "./src/ui/**/*.{js,jsx,ts,tsx}",
  ],
  plugins: [
    require("@tailwindcss/container-queries"),
  ],
}
```

<Warning>
The `@tailwindcss/container-queries` plugin is required. Install it with your package manager.
</Warning>

### For Tailwind v4

Import the theme in your global CSS file:

```css src/app/globals.css
@import "tailwindcss";
@import "../ui/theme.css";
```

## Step 7: Sync components

Run the sync command to download components:

```bash
npx @subframe/cli@latest sync --all
```

The CLI reads your `.subframe` directory settings and populates your project with component code.

## Step 8: Import fonts (if needed)

If your theme uses custom fonts, add them to your HTML:

### Next.js

```tsx app/layout.tsx
import { Inter } from 'next/font/google'

const inter = Inter({ subsets: ['latin'] })

export default function RootLayout({ children }: { children: React.Node }) {
  return (
    <html className={inter.className}>
      <body>{children}</body>
    </html>
  )
}
```

### Vite or other frameworks

Add to your `index.html`:

```html
<head>
  <link rel="preconnect" href="https://fonts.googleapis.com">
  <link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>
  <link href="https://fonts.googleapis.com/css2?family=Inter:wght@400;500;600;700&display=swap" rel="stylesheet">
</head>
```

## Step 9: Verify installation

Test that everything works:

```tsx src/App.tsx
import { Button } from "@/ui/Button"

export default function App() {
  return <Button>Click me</Button>
}
```

Start your dev server and verify the button renders with styling.

## Troubleshooting

### Import errors

If imports don't resolve:

1. Check `tsconfig.json` paths configuration
2. Verify the import alias matches your path aliases
3. Restart your TypeScript server
4. Check that files exist at the expected paths

### Styles not applying

If Tailwind styles don't work:

1. Verify Tailwind CSS is imported in your app
2. Check `content` paths include `src/ui/**`
3. Make sure `@tailwindcss/container-queries` is installed
4. Clear build cache and restart dev server

### Component errors

If components have runtime errors:

1. Verify `@subframe/core` is installed
2. Check that you copied the complete component code
3. Make sure all imports are correct
4. Look for missing dependencies in component code

### Type errors

If TypeScript errors appear:

1. Check that component interfaces are exported
2. Verify React types are installed: `@types/react`
3. Make sure `tsconfig.json` includes component files
4. Restart TypeScript server in your editor
