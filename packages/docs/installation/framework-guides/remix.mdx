Subframe works with Remix's server-side rendering and client-side hydration. Since Subframe components are interactive React components, you'll use them in client-side code with proper hydration.

## Prerequisites

You need a Remix project with:
- Tailwind CSS configured
- TypeScript set up (recommended)
- Node.js 18+ installed

If you're starting fresh, create a new Remix project:

```bash
npx create-remix@latest
```

Choose the Remix App Server and TypeScript when prompted. Make sure to enable Tailwind CSS during setup.

## Installation

Run the Subframe CLI in your Remix project directory:

<CodeGroup>
```bash npm
npx @subframe/cli@latest init
```

```bash yarn
yarn dlx @subframe/cli@latest init
```

```bash pnpm
pnpx @subframe/cli@latest init
```

```bash bun
bunx @subframe/cli@latest init
```
</CodeGroup>

The CLI will:
1. Prompt for your access token from [app.subframe.com/cli/auth](https://app.subframe.com/cli/auth)
2. Detect your Tailwind version and configure it
3. Set up the component sync directory (default: `./app/ui`)
4. Configure TypeScript path aliases
5. Optionally sync all components

## Remix integration

### Using components in routes

Subframe components work in any Remix route. Import them using the configured alias:

```tsx app/routes/_index.tsx
import { Button } from "@/ui/Button"
import { TextField } from "@/ui/TextField"

export default function Index() {
  return (
    <div>
      <h1>Welcome</h1>
      <TextField.Root>
        <TextField.Input placeholder="Enter your email" />
      </TextField.Root>
      <Button>Get Started</Button>
    </div>
  )
}
```

All Subframe components include `"use client"` directives and work with Remix's hydration automatically.

### Forms and actions

Use Subframe components with Remix's form handling:

```tsx app/routes/contact.tsx
import { Form, useActionData, useNavigation } from "@remix-run/react"
import { Button } from "@/ui/Button"
import { TextField } from "@/ui/TextField"
import { Alert } from "@/ui/Alert"

export async function action({ request }: ActionArgs) {
  const formData = await request.formData()
  const email = formData.get("email")

  // Handle form submission
  return { success: true }
}

export default function Contact() {
  const actionData = useActionData<typeof action>()
  const navigation = useNavigation()
  const isSubmitting = navigation.state === "submitting"

  return (
    <Form method="post">
      <TextField.Root>
        <TextField.Label>Email</TextField.Label>
        <TextField.Input name="email" type="email" required />
      </TextField.Root>

      <Button type="submit" disabled={isSubmitting}>
        {isSubmitting ? "Sending..." : "Submit"}
      </Button>

      {actionData?.success && (
        <Alert variant="success">Form submitted successfully!</Alert>
      )}
    </Form>
  )
}
```

### Navigation with Remix Link

Replace button `onClick` navigation with Remix's `Link` component:

```tsx
import { Link } from "@remix-run/react"
import { Button } from "@/ui/Button"

export default function HomePage() {
  return (
    <Link to="/dashboard">
      <Button>Go to Dashboard</Button>
    </Link>
  )
}
```

Or wrap Link around Subframe components directly:

```tsx
import { Link } from "@remix-run/react"
import { Card } from "@/ui/Card"

<Link to="/product/123">
  <Card>
    <Card.Title>Product Name</Card.Title>
    <Card.Description>Product details</Card.Description>
  </Card>
</Link>
```

## Styling configuration

### Tailwind setup

The CLI automatically configures Tailwind. Your `tailwind.config.ts` should include:

```ts tailwind.config.ts
import type { Config } from "tailwindcss"

export default {
  presets: [require("./app/ui/tailwind.config.js")],
  content: [
    "./app/**/*.{js,jsx,ts,tsx}",
    "./app/ui/**/*.{js,jsx,ts,tsx}",
  ],
} satisfies Config
```

The preset file (`app/ui/tailwind.config.js`) contains your theme tokens from Subframe.

### CSS imports

Make sure your root route imports Tailwind. The CLI prompts for this path during init (typically `app/styles/tailwind.css` or `app/tailwind.css`):

```tsx app/root.tsx
import type { LinksFunction } from "@remix-run/node"
import stylesheet from "./tailwind.css"

export const links: LinksFunction = () => [
  { rel: "stylesheet", href: stylesheet },
]

export default function App() {
  return (
    <html>
      <head>
        <Links />
      </head>
      <body>
        <Outlet />
        <Scripts />
      </body>
    </html>
  )
}
```

## Server-side rendering

Subframe components render on the server and hydrate on the client automatically. All components support SSR out of the box.

### Client-only patterns

For components with browser-only APIs (like those using `window` or `localStorage`), wrap them in Remix's `ClientOnly`:

```tsx
import { ClientOnly } from "remix-utils/client-only"
import { SomeClientComponent } from "./SomeClientComponent"

<ClientOnly fallback={<div>Loading...</div>}>
  {() => <SomeClientComponent />}
</ClientOnly>
```

But most Subframe components don't need thisâ€”they handle SSR correctly.

## Syncing components

After updating components in Subframe, sync them to your Remix app:

```bash
npx @subframe/cli sync --all
```

The CLI updates component files in your configured directory (`app/ui` by default). Remix's dev server auto-reloads with the changes.

## Best practices

**Use Remix's built-in features** - Leverage Remix's Form, Link, and useNavigation instead of recreating functionality with Subframe components.

**Progressive enhancement** - Subframe components work without JavaScript, but add event handlers for enhanced experiences. Remix forms still work if JS fails to load.

**Server-side data loading** - Fetch data in Remix loaders, not in components. Pass data as props to Subframe components.

**Organize components** - Keep page-specific logic separate from Subframe UI components. Create wrapper components in `app/components/` when needed.

## Troubleshooting

### Styles not applying

If Tailwind styles don't work:

1. Verify `tailwind.css` is imported in your root route
2. Check that `content` paths in `tailwind.config.ts` include `app/ui/**`
3. Restart the dev server: `npm run dev`

### Hydration mismatches

If you see hydration errors:

1. Make sure components don't use browser-only APIs during render
2. Check that server and client render the same initial HTML
3. Use `ClientOnly` wrapper for truly client-side components

### Import errors

If imports don't resolve:

1. Check `tsconfig.json` has path aliases configured
2. Verify the alias matches what you configured during init (default: `@/ui/*`)
3. Restart your TypeScript server in your editor
