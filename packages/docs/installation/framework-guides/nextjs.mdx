## Installation

Run the init command in your repository root:

<CodeGroup>
  ```bash npm
  npx @subframe/cli@latest init
  ```
  ```bash yarn
  yarn dlx @subframe/cli@latest init
  ```
  ```bash pnpm
  pnpx @subframe/cli@latest init
  ```
  ```bash bun
  bunx --bun @subframe/cli@latest init
  ```
</CodeGroup>

The CLI detects Next.js and configures everything automatically.

## App Router (Next.js 13+)

Subframe components work with both client and server components.

### Client components

Most Subframe components need client-side interactivity:

```tsx app/page.tsx
"use client";

import { Button } from "@/ui/Button";

export default function Page() {
  return (
    <Button onClick={() => console.log("clicked")}>
      Click me
    </Button>
  );
}
```

### Server components

Layout primitives without interactivity work in server components:

```tsx app/layout.tsx
import { Container } from "@/ui/Container";

export default function RootLayout({ children }) {
  return (
    <html>
      <body>
        <Container>{children}</Container>
      </body>
    </html>
  );
}
```

## Pages Router (Next.js 12 and earlier)

Import and use components in your pages:

```tsx pages/index.tsx
import { Button } from "@/ui/Button";

export default function Home() {
  return <Button>Click me</Button>;
}
```

## Navigation with Next.js Link

Use Next.js `Link` for navigation:

```tsx
import Link from "next/link";
import { Button } from "@/ui/Button";

export default function Page() {
  return (
    <Link href="/about">
      <Button>Go to About</Button>
    </Link>
  );
}
```

## Image optimization

Use `next/image` with Subframe image slots:

```tsx
import Image from "next/image";
import { Card } from "@/ui/Card";

export default function Page() {
  return (
    <Card
      image={
        <Image
          src="/hero.jpg"
          alt="Hero image"
          width={800}
          height={600}
        />
      }
    >
      Card content
    </Card>
  );
}
```
