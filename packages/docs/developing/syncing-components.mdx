Syncing creates a local copy of your Subframe components that stays in sync with your designs. Components are your design system source of truthâ€”they sync bidirectionally via CLI.

**Components sync, pages don't.** This is the fundamental workflow distinction:
- **Components** = Run `sync` command to update code when designs change
- **Pages** = Copy/paste code once, then add business logic locally (no syncing)

<img src="https://developers.elementor.com/docs/assets/img/elementor-placeholder-image.png" alt="Terminal showing the Subframe CLI sync command running with output displaying synced components like Button, TextField, and Card with success checkmarks" className="w-full" />

<Note>
Sync goes one way from Subframe to your codebase. Local changes to component files get overwritten on the next sync. Use wrapper components for custom logic.
</Note>

## Running sync

Make sure to first [install Subframe](/installation/existing-project). Sync all components with:

<CodeGroup>
  ```bash npm
  npx @subframe/cli@latest sync --all
  ```
  ```bash yarn
  yarn dlx @subframe/cli@latest sync --all
  ```
  ```bash pnpm
  pnpx @subframe/cli@latest sync --all
  ```
  ```bash bun
  bunx --bun @subframe/cli@latest sync --all
  ```
</CodeGroup>

### Sync specific components

Provide component names as arguments:

<CodeGroup>
  ```bash npm
  npx @subframe/cli@latest sync Alert Accordion
  ```
  ```bash yarn
  yarn dlx @subframe/cli@latest sync Alert Accordion
  ```
  ```bash pnpm
  pnpx @subframe/cli@latest sync Alert Accordion
  ```
  ```bash bun
  bunx --bun @subframe/cli@latest sync Alert Accordion
  ```
</CodeGroup>

### Command options

```
Arguments:
  components                   the components to sync

Options:
  -a, --all                    sync all components
  -p, --projectId <projectId>  project id to run sync with
  -i, --install                install dependencies after syncing
  -h, --help                   display help for command
```

Sync all components and install dependencies:

```bash
npx @subframe/cli@latest sync --all --install
```

## When to sync

Sync your components whenever you:

- **Create new components** in Subframe to use in your codebase
- **Modify existing components** (change styles, add variants, update props)
- **Pull changes** from teammates who updated components
- **Update your theme** (colors, typography, spacing)
- **Add new variants** or props to components
- **Fix bugs** in component implementations

Sync only updates files that changed. Run it as often as needed.

After syncing, restart your dev server to see changes:

```bash
npm run dev
```

## Preventing overwrites

If you need to modify a component file directly, add `// @subframe/sync-disable` to prevent overwrites. See [Disabling sync](/developing/disabling-sync) for details and best practices.

<Warning>
Disabling sync should be a last resort. Use wrapper components or composition patterns when possible.
</Warning>
