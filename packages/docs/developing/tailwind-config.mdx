Subframe extends your Tailwind CSS configuration with design tokens from your theme. The CLI adds colors, typography, spacing, and utilities via a preset.

## How it works

When you run `npx @subframe/cli init`, the CLI modifies your `tailwind.config.js` or `tailwind.config.ts` file. It adds a Tailwind preset that includes Subframe's design tokens—your existing configuration stays intact.

The CLI adds:

- **Colors** - Theme palette (brand, neutral, error, warning, success)
- **Typography** - Font families, sizes, and weights
- **Spacing** - Custom spacing scale
- **Border radius** - Consistent radius values
- **Box shadows** - Elevation tokens
- **Breakpoints** - Mobile breakpoint (767px and below)
- **Content paths** - Ensures Tailwind scans Subframe component files

Your config file looks like this after installation:

```ts tailwind.config.ts
import type { Config } from "tailwindcss"

const config: Config = {
  content: [
    "./src/**/*.{js,ts,jsx,tsx,mdx}",
    "./src/ui/**/*.{tsx,ts,js,jsx}",  // Synced Subframe components
  ],
  presets: [require("./src/ui/tailwind.config.js")],  // Subframe theme tokens
  plugins: [
    require("@tailwindcss/container-queries"),  // Required
  ],
}

export default config
```

When you update your theme in Subframe and run `npx @subframe/cli sync`, the CLI updates token values automatically.

## Using theme tokens

All tokens from your Subframe theme are available as Tailwind classes.

**Colors** use the full 50-950 scale:

```tsx
<div className="bg-brand-100 text-brand-900">
  Brand colors
</div>

<div className="bg-error-500 text-white">
  Error state
</div>
```

**Typography** includes your theme fonts and sizes:

```tsx
<h1 className="font-sans text-4xl font-bold">
  Heading
</h1>

<p className="text-base font-normal">
  Body text
</p>
```

**Spacing** follows your theme scale (typically 4, 8, 12, 16, 20, 24, 32, 40, 48, 64, 80, 96px):

```tsx
<div className="p-6 gap-4 m-8">
  Theme spacing
</div>
```

**Borders and shadows** match your design system:

```tsx
<div className="rounded-md border border-neutral-200 shadow-lg">
  Consistent styling
</div>
```

## Customization

You can add your own Tailwind configuration alongside Subframe's tokens. The preset approach means you can extend the theme in your root config:

```ts tailwind.config.ts
const config: Config = {
  presets: [require("./src/ui/tailwind.config.js")],  // Subframe preset
  theme: {
    extend: {
      // Add custom values alongside Subframe tokens:
      colors: {
        custom: {
          purple: "#8b5cf6",
        },
      },
      spacing: {
        '128': '32rem',
      },
    },
  },
}
```

### Content paths

Keep both content paths in your configuration:

```ts
content: [
  "./src/**/*.{js,ts,jsx,tsx,mdx}",     // Your files
  "./src/ui/**/*.{tsx,ts,js,jsx}",      // Synced Subframe components
],
```

Add more paths for components in other directories:

```ts
content: [
  "./src/**/*.{js,ts,jsx,tsx,mdx}",
  "./app/**/*.{js,ts,jsx,tsx,mdx}",     // Next.js app directory
  "./src/ui/**/*.{tsx,ts,js,jsx}",      // Synced Subframe components
],
```

### Responsive design

Subframe adds a `mobile:` breakpoint that activates at 767px and below:

```tsx
<div className="flex flex-row mobile:flex-col gap-6 mobile:gap-4">
  {/* Horizontal on desktop, vertical on mobile */}
</div>
```

This matches how you set mobile overrides in the Subframe editor. Tailwind's default breakpoints (`sm:`, `md:`, `lg:`) still work—use whichever fits your needs.

### Plugins

Keep `@tailwindcss/container-queries` in your plugins array—Subframe components require it. Add other plugins as needed:

```ts
plugins: [
  require("@tailwindcss/container-queries"),  // Required
  require("@tailwindcss/forms"),
  require("@tailwindcss/typography"),
],
```

### Tailwind v4

Subframe supports both Tailwind CSS v3 and v4. The CLI detects your version and generates appropriate configuration automatically.

**Tailwind v3** uses `tailwind.config.js` or `tailwind.config.ts` files.

**Tailwind v4** uses CSS-based configuration with `@theme` directives. The CLI generates theme tokens as CSS variables.

## Common issues

<AccordionGroup>
<Accordion title="Styles not applying">

If Tailwind classes don't work:

1. Verify the content paths include your files and `./src/ui/**/*.{tsx,ts,js,jsx}`
2. Check that Tailwind CSS is imported in your app
3. Restart your dev server

</Accordion>
<Accordion title="Build errors">

If you see Tailwind build errors:

1. Check for syntax errors in `tailwind.config`
2. Verify `@tailwindcss/container-queries` is installed: `npm install @tailwindcss/container-queries`
3. Clear `node_modules` and reinstall if needed

</Accordion>
<Accordion title="Classes not purged correctly">

If production builds include unused classes:

1. Verify content paths cover all files using Tailwind
2. Use full class names: `"bg-brand-500"` not `"bg-" + color`
3. Don't use string concatenation for class names

</Accordion>
<Accordion title="Mobile breakpoint not working">

If `mobile:` classes don't apply:

1. Verify the mobile breakpoint exists in your config (max-width: 767px)
2. Test in browser dev tools with responsive mode
3. Use `mobile:` prefix, not `sm:`, for Subframe's mobile breakpoint

</Accordion>
</AccordionGroup>
