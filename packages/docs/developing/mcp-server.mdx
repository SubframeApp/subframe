The Subframe MCP server connects AI assistants like Claude Code and Cursor to your Subframe projects. AI fetches components, pages, and themes without manual code copying.

[Model Context Protocol (MCP)](https://modelcontextprotocol.io/docs/getting-started/intro) is a standard for connecting AI assistants to external data sources. A separate Subframe Docs MCP server provides access to documentation.

## Installation

The easiest way to install is from Subframe:

1. Open a page in Subframe
2. Open Code mode in the top right
3. Click the Installation tab
4. Follow instructions for Cursor or Claude Code

For manual installation, get an authentication token at [app.subframe.com/settings/tokens](https://app.subframe.com/settings/tokens).

<Tabs>
<Tab title="Cursor">

<Steps>
<Step title="One-click install (recommended)">
Click **Add to Cursor** in the Code Mode Installation tab to configure Cursor with your MCP servers.
</Step>
<Step title="Manual configuration">
Add to your Cursor MCP configuration (`~/.cursor/mcp.json`):

```json
{
  "mcpServers": {
    "subframe": {
      "url": "https://mcp.subframe.com/mcp",
      "headers": {
        "Authorization": "Bearer YOUR_AUTH_TOKEN"
      }
    },
    "subframe-docs": {
      "url": "https://docs.subframe.com/mcp"
    }
  }
}
```
</Step>
<Step title="Verify installation">
Ask Cursor "What tools do you have available?" to verify MCP servers are connected.
</Step>
</Steps>

This adds two servers: `subframe` for your projects and `subframe-docs` for documentation.

</Tab>
<Tab title="Claude Code">

<Steps>
<Step title="Install the Subframe MCP server">

```bash
claude mcp add --transport http subframe https://mcp.subframe.com/mcp \
  --header "Authorization: Bearer YOUR_AUTH_TOKEN"
```
</Step>
<Step title="Install Subframe Docs MCP (optional)">

```bash
claude mcp add --transport http subframe-docs https://docs.subframe.com/mcp
```
</Step>
</Steps>

</Tab>
<Tab title="Other MCP clients">

<Steps>
<Step title="Configure Subframe MCP Server">
Configure your MCP client:

- **URL:** `https://mcp.subframe.com/mcp`
- **Transport:** HTTP
- **Authentication:** Bearer token
- **Header:** `Authorization: Bearer YOUR_AUTH_TOKEN`
</Step>
<Step title="Configure Subframe Docs MCP (optional)">
Optionally add documentation access:

- **URL:** `https://docs.subframe.com/mcp`
- **Transport:** HTTP
- **Authentication:** None required
</Step>
<Step title="Verify installation">
Restart your MCP client and verify servers are available.
</Step>
</Steps>

</Tab>
</Tabs>

## Using the MCP server

Your AI assistant accesses Subframe when you prompt with an MCP link from the Code panel.

### Available tools

| Tool | Description | Input Parameters | Returns |
|------|-------------|-----------------|---------|
| `list_components` | Lists all components in your project | `projectId` (optional) | Array of components with `id`, `name`, `url` |
| `list_pages` | Lists all pages in your project | `projectId` (optional) | Array of pages with `id`, `name`, `url` |
| `get_component_info` | Gets code and metadata for a component | `id`, `name`, or `url`<br/>`projectId` (optional) | `id`, `name`, `files` |
| `get_page_info` | Gets code for a page | `id`, `name`, or `url`<br/>`projectId` (optional) | `id`, `name`, `files` |
| `get_theme` | Generates Tailwind theme for the project | `projectId` (optional)<br/>`cssType` (optional) | `theme` config |

If `projectId` is not specified, the tool uses the first project you have access to.

### Prompt with MCP link

Use the MCP link from the Code Inspect panel:

```text
Implement the design at https://app.subframe.com/design/...
```

<Note>
To get the latest version of components, run `npx @subframe/cli@latest sync` to [sync components](/developing/syncing-components).
</Note>

## Troubleshooting

<AccordionGroup>
<Accordion title="Authentication failed">

Check that:
- Your API token is correct and hasn't expired
- You copied the entire token without extra spaces
- The token is in the `Authorization` header as `Bearer YOUR_AUTH_TOKEN`

Generate a new token at [app.subframe.com/settings/tokens](https://app.subframe.com/settings/tokens).

</Accordion>
<Accordion title="AI not calling the server">

Make sure your AI tool:
- Has MCP support enabled
- Has the Subframe server in its MCP configuration
- Has been restarted after adding configuration

Check your tool's logs for MCP-related errors.

</Accordion>
<Accordion title="Connection errors">

If the MCP server is unreachable:
- Check your internet connection
- Verify the URL is `https://mcp.subframe.com/mcp`
- Contact Subframe support if issues persist

</Accordion>
</AccordionGroup>
