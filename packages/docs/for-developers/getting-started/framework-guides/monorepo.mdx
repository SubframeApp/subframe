If you just want to get setup with Vite and Subframe quickly, you can follow the [quickstart guide for Vite](/for-developers/getting-started/quickstart#vite). If you want to set up Vite manually, you can follow the steps below.

## Set up your Vite app

<Note>
    You can skip this step if you already have a Vite app set up. 
    The important part here is having your Vite app set up to use React and Typescript.
</Note>
<Info>
For more information on setting up your Vite app, you can visit the [Vite documentation](https://vitejs.dev/guide/).
</Info>

<CodeGroup>
    ```bash npm
    npm create vite@latest -- --template react-ts
    ```
    ```bash yarn
    yarn create vite --template react-ts
    ```
    ```bash pnpm
    pnpm create vite --template react-ts
    ```
    ```bash bun
    bun create vite --template react-ts
    ```
</CodeGroup>


## Set up TailwindCSS

<Note>
    You can skip this step if you already have TailwindCSS set up in your Vite app. Make sure to [include Subframe's preset in your TailwindCSS config](#run-the-subframe-cli) however.
</Note>
<Info>
    For more information on setting up TailwindCSS, you can visit the [TailwindCSS documentation](https://v3.tailwindcss.com/docs/installation).
</Info>

Subframe components use TailwindCSS for styling. In this step, we'll set up TailwindCSS in your Vite app.

<Steps>
    <Step title="Install TailwindCSS, PostCSS and Autoprefixer">
        <Warning>
            As of now, Subframe is dependent on TailwindCSS' ability to use presets to extend the base Tailwind Config with Subframe's. This feature is not available in TailwindCSS v4.0 and above (at the time of writing), so we need to make sure we're using TailwindCSS@^3.0.0.
        </Warning>

        <CodeGroup>
            ```bash npm
            npm install -D tailwindcss@^3.0.0 postcss autoprefixer
            ```
            ```bash yarn
            yarn add -D tailwindcss@^3.0.0 postcss autoprefixer
            ```
            ```bash pnpm
            pnpm add -D tailwindcss@^3.0.0 postcss autoprefixer
            ```
            ```bash bun
            bun add -D tailwindcss@^3.0.0 postcss autoprefixer
            ```
        </CodeGroup>
    </Step>

    <Step title="Initialize your TailwindCSS config">
        <CodeGroup>
            ```bash npm
            npx tailwindcss init -p
            ```
            ```bash yarn
            yarn tailwindcss init -p
            ```
            ```bash pnpm
            pnpm exec tailwindcss init -p
            ```
            ```bash bun
            bunx tailwindcss init -p
            ```
        </CodeGroup>

        There should now be a `postcss.config.js` and a `tailwind.config.js` file in your project root. The former you can leave as-is, we will now change the latter to include our root `index.html` file like so:

        ```javascript tailwind.config.js {4}
        /** @type {import('tailwindcss').Config} */
        export default {
            content: [
                './index.html',
            ],
            theme: {
                extend: {},
            },
            plugins: [],
        }
        ```

</Step>
<Step title="Add Tailwind to your CSS">
        In your `src/index.css` file, add the following:

        ```css src/index.css {1-3}
        @tailwind base;
        @tailwind components;
        @tailwind utilities;
        ```

        This will include the base, components and utilities styles from TailwindCSS.
    </Step>
</Steps>

Tailwind should now be set up in your Vite app and we're ready to move on to installing Subframe.


## Initialize Subframe

To be able to sync our components within our Vite app and use them, we will need to initialize the project and configure it to be able to style Subframe.

The easiest way of doing so is the Subframe CLI, which will initialize the project for you and set up some necessary configurations.


<Steps>
    <Step title="Run the Subframe CLI">
        The following command will set up Subframe in your project. Replace `[PROJECT_ID]` with the your project's ID on Subframe. 
        You can find your project's ID by [opening your project in the Subframe App](https://app.subframe.com/) and looking at the URL: `https://app.subframe.com/[PROJECT_ID]/library`.

        <CodeGroup>
            ```bash npm
            npx @subframe/cli@latest init -p [PROJECT_ID]
            ```
            ```bash yarn
            yarn @subframe/cli@latest init -p [PROJECT_ID]
            ```
            ```bash pnpm
            pnpm exec @subframe/cli@latest init -p [PROJECT_ID]
            ```
            ```bash bun
            bunx @subframe/cli@latest init -p [PROJECT_ID]
            ```
        </CodeGroup>

        <Steps>
            <Step title="Authenticate">
                Depending on whether you've used the Subframe CLI before, you'll be asked to authenticate when initializing your project. Simply follow the instructions in the terminal to authenticate.
            </Step>
            <Step title="Create .subframe directory">
                The CLI will ask you if it should create a `.subframe` directory in your project. Press `y` to confirm.
            </Step>
            <Step title="Configure Subframe components directory">
                In most cases, you will want your Subframe components to be synced to `./src` in your project. This will result in your components being synced to `src/subframe` in your project's codebase. If you want to sync your components to a different directory, you can specify so now.
            </Step>
            <Step title="Configure import alias">
                Next, the CLI will set up an import alias for `@/subframe` for you. You can define what kind of alias you prefer here. Afterwards, the CLi will ask if you would like to add an alias to `tsconfig.json`. Since Vite will not directly play nicely with this, you can press `n` here. We'll fix this later.
            </Step>
            <Step title="Install dependencies">
                The CLI will ask if it should install dependencies for you. If you're using npm, you can press `y` and skip the following, if you're using a different package manager, press `n` and run the following command depending on your package manager:

                <CodeGroup>
                    ```bash yarn
                    yarn add @subframe/core classnames
                    ```
                    ```bash pnpm
                    pnpm add @subframe/core classnames
                    ```
                    ```bash bun
                    bun add @subframe/core classnames
                    ```
                </CodeGroup>
            </Step>
            <Step title="Configure Tailwind">
                Finally, the CLI will ask if it should configure your Tailwind config. Press `y` to confirm. This will add the necessary configurations to your `tailwind.config.js` file. Your `tailwind.config.js` file should now look like this:
            </Step>
        </Steps>

        ```javascript tailwind.config.js {5, 12}
        /** @type {import('tailwindcss').Config} */
        export default {
            content: [
                "./index.html",
                "./src/subframe/**/*.{tsx,ts,js,jsx}" 
            ],
            theme: {
                extend: {},
            },
            plugins: [],
            presets: [
                require("./src/subframe/tailwind.config.js")
            ]
        };
        ```
    </Step>

    <Step title="Configure Vite to use import aliases">

        Vite doesn't directly play nicely with TypeScript import aliases, so we'll need to set up a plugin to handle this for us.

        First, install the `vite-tsconfig-paths` package:

        <CodeGroup>
            ```bash npm
            npm install -D vite-tsconfig-paths
            ```
            ```bash yarn
            yarn add -D vite-tsconfig-paths
            ```
            ```bash pnpm
            pnpm add -D vite-tsconfig-paths
            ```
            ```bash bun
            bun add -D vite-tsconfig-paths
            ```
        </CodeGroup>

        Next, add the plugin to your `vite.config.ts` file:

        ```javascript vite.config.ts {3, 9}
        import { defineConfig } from 'vite'
        import react from '@vitejs/plugin-react'
        import tsconfigPaths from 'vite-tsconfig-paths'

        // https://vite.dev/config/
        export default defineConfig({
            plugins: [
                react(),
                tsconfigPaths(),
            ],
        })
        ```

        Now configure you import alias in `tsconfig.app.json`. You'll need to provide values for `baseUrl`, `allowJS` [(see why)](https://github.com/aleclarson/vite-tsconfig-paths?tab=readme-ov-file#%EF%B8%8F-non-typescript-modules-need-special-configuration) and `paths`.

        ```json tsconfig.app.json {3-9}
        {
            "compilerOptions": {
                "baseUrl": "./",
                "allowJs": true,
                "paths": {
                    "@/*": [
                        "./src/*"
                    ]
                },
                "tsBuildInfoFile": "./node_modules/.tmp/tsconfig.app.tsbuildinfo",
                "target": "ES2020",
                "useDefineForClassFields": true,
                "lib": ["ES2020", "DOM", "DOM.Iterable"],
                "module": "ESNext",
                "skipLibCheck": true,

                /* Bundler mode */
                "moduleResolution": "bundler",
                "allowImportingTsExtensions": true,
                "isolatedModules": true,
                "moduleDetection": "force",
                "noEmit": true,
                "jsx": "react-jsx",

                /* Linting */
                "strict": true,
                "noUnusedLocals": true,
                "noUnusedParameters": true,
                "noFallthroughCasesInSwitch": true,
                "noUncheckedSideEffectImports": true+
            },
            "include": ["src"]
        }
        ```
    </Step>
</Steps>

You now have Subframe set up to use in your Vite app. You can now go ahead and sync your components to use them in your project.
