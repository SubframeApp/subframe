## Who uses Subframe?

**Designers** create production-ready UI without handoffs. The visual editor feels familiar, but what you build is real React code that syncs to your codebase.

**Developers** skip pixel-pushing and focus on business logic. Components sync automatically from Subframe—design changes update your code via CLI.

**Product builders** go from idea to working prototype in minutes. Generate designs with AI, then export to Cursor, Bolt, or Replit to add logic.

## How it works

Subframe is both a full design tool and your source of truth for production code. When you design, you're working with real React components—not mockups.

Components sync one-way to your codebase via CLI. Change the design, run `sync`, your code updates. Pages export as React you copy and paste. Your theme syncs as Tailwind tokens. Everything lives locally in your codebase for further development.

## What makes it different

**Visual editing meets production code** — Drag-and-drop editor using flexbox layouts and responsive design. Everything exports as clean React + Tailwind, not AI-generated code.

**AI that learns your design system** — Generate pages from prompts or images. AI uses your theme and components to stay on-brand. Mix and match variations or refine in the editor.

**True design system sync** — Components you design are the components engineers use. Props, variants, slots, TypeScript types—synced to your codebase as the single source of truth.
