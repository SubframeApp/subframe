## Projects

Your Subframe project contains your theme, components, and pages. Each project is an independent design system—the source of truth for your product UI.

<img src="https://developers.elementor.com/docs/assets/img/elementor-placeholder-image.png" alt="The Project Switcher dropdown showing a list of projects with a New project button" />

## Components

Reusable UI elements that sync bidirectionally between Subframe and your codebase. Components are your design system source of truth—design once, sync everywhere.

**Two-way sync:** Run `npx @subframe/cli sync` to push component changes to your code. Props, variants, slots, and TypeScript types update automatically. Subframe owns the design, you own the integration.

**Built-in library:** Subframe includes components built on Radix and Headless UI. Every component is real React code with full documentation. Use them as-is or create custom components.

## Pages

Individual screens you design with components. Pages export as React code you copy and paste into your project—one-way export, no reverse sync.

**Copy/paste workflow:** Generate pages with AI or build manually, then export code. Paste into your project and add business logic. Developers own the page after export—no syncing back to Subframe.

**Best for integration:** Pages let you mix Subframe designs with custom code, API calls, state management, and routing. Components stay synced, pages stay flexible.

## Ask AI

Generate designs from text prompts or images. AI learns from your designs to create on-brand pages using your theme and components—you curate and remix variations.

**Context hierarchy:** AI personalization builds from multiple layers:
- **Theme** — Colors, typography, spacing define your visual language
- **Components** — Your design system becomes AI's building blocks
- **System prompts** — Embed product context, design guidelines, brand voice
- **Existing pages** — Each page you create trains the AI further

The more you build in Subframe, the smarter your AI becomes. Everything generated stays fully editable.

## Prototypes

Interactive prototypes with working navigation and logic. Add annotations to your design—comments become prompts that drive behavior. Design is the best prompt.

**Annotations-based workflow:** After designing, add comments describing interactions. AI generates working prototypes that match your design exactly. No chat required—annotations capture your intent directly on the design.

**Design-as-prompt:** Your visual design already describes functionality. Buttons, forms, navigation—all context for the AI. Annotations add the "what happens when" layer.

**Rebuild anytime:** Annotations persist as inline code comments. Update your design, regenerate the prototype, and get the same results. LLMs become the new programming language.

## MCP server

Connect AI coding tools like Claude Code and Cursor to your Subframe projects. AI assistants fetch components and pages directly—paste Subframe URLs into prompts and the MCP server handles everything.

**Available tool calls:** MCP server provides tools to get project info, component metadata, page code, and theme configuration. Your designs become the best context for your AI coding assistant.

**Multi-page context:** Paste multiple Subframe URLs to give AI assistant access to entire flows. Better than copying code manually—AI sees relationships between pages and components.

**Preserves design integrity:** AI fetches exact Subframe code, then adds business logic on top. Your design stays perfect while you focus on functionality.
